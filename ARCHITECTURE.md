# WAAP 智能规则生成工具架构设计

## 1. 项目概述

本项目旨在构建一个基于大模型的 WAAP/WAF 防护规则生成系统，通过分析攻击样本自动生成高质量的防护规则。

## 2. 核心功能模块

### 2.1 规则生成模块
- **功能**：调用 LLM API 生成防护规则
- **支持的攻击类型**：SQL 注入、XSS、RCE、CSRF 等多种攻击类型
- **核心流程**：
  - 分析攻击样本特征
  - 构建提示词引导大模型生成规则
  - 提取和处理生成的规则

### 2.2 规则验证模块
- **功能**：验证生成规则的有效性
- **核心流程**：
  - 语法验证
  - 匹配率测试
  - 误报率测试
  - 性能测试

### 2.3 大模型接口模块
- **功能**：封装 API 调用逻辑
- **核心特性**：
  - API 调用重试和错误处理
  - 支持 GLM-5、GPT 等模型
  - 优化提示词减少 token 消耗
  - 缓存机制

### 2.4 攻击类型识别模块
- **功能**：自动识别攻击类型
- **支持的类型**：
  - SQL 注入 (SQLi)
  - 跨站脚本 (XSS)
  - 远程代码执行 (RCE)
  - 命令注入 (Command Injection)
  - 跨站请求伪造 (CSRF)
  - 其他类型

## 4. 命令行接口

### 4.1 生成规则
```bash
waap generate -i <input> -o <output> -t <type>
```
- `-i, --input`：输入攻击样本
- `-o, --output`：输出文件
- `-t, --type`：攻击类型
- `--format`：输出格式 (regex, keyword, combined)

### 4.2 验证规则
```bash
waap validate -r <rule> -s <sample> -n <negative>
```
- `-r, --rule`：要验证的规则
- `-s, --sample`：正面样本
- `-n, --negative`：负面样本
- `--report`：生成报告

### 4.3 批量处理
```bash
waap batch -d <directory> -o <output>
```
- `-d, --directory`：样本目录
- `-o, --output`：输出目录

### 4.4 配置管理
```bash
waap config [command]
```
- `set`：设置配置
- `get`：获取配置
- `reset`：重置配置

## 6. 工作流程

1. **输入处理**：接收攻击样本
2. **类型识别**：识别攻击类型
3. **规则生成**：调用 LLM 生成规则
4. **规则验证**：验证规则有效性
5. **规则优化**：根据验证结果优化
6. **输出规则**：保存或展示生成的规则

## 7. 技术优势

- **智能生成**：利用大模型生成高质量规则
- **多模型支持**：兼容 LLM 模型
- **规则导出**：支持导出为 ModSecurity、NGINX、Cloudflare 等格式
- **缓存机制**：优化 API 调用，提高性能
